<!-- Schema from data/exercises/gap-fill.json (existing CE exercise files) -->
<!-- Syntax exercise files follow this schema exactly. -->
<!-- "sentence" = the full Latin sentence (for reference). -->
<!-- "display" = what is shown to the student — for ID questions show full sentence; for translation questions show full sentence. -->
<!-- "translation" = English context hint shown beneath the question in the engine. -->
<!-- "options" = array of exactly 4 strings. -->
<!-- "correct" = must exactly match one of the 4 strings in options. -->
<!-- "explanation" = shown after every answer, correct or not. -->

Example from data/exercises/gap-fill.json:
{
  "id": "gf-01",
  "sentence": "puella in viā ambulat.",
  "blank_word": "viā",
  "display": "puella in _____ ambulat.",
  "translation": "The girl walks in the street.",
  "options": ["viā", "silvā", "aquā", "portā"],
  "correct": "viā",
  "explanation": "'viā' (road/street, ablative) follows 'in'. The girl walks in/along the street."
}

For SYNTAX exercises there is no blank_word (not a gap-fill). Use these adaptations:
- ID question: display = full Latin sentence; translation = English gloss of the sentence; options = 4 construction names; correct = the construction being used; explanation = why this construction applies.
- Translation question: display = full Latin sentence; translation = "" (empty — student must translate); options = 4 English translations; correct = the accurate translation; explanation = how the construction maps to English.
- Set blank_word to "" (empty string) for all syntax questions — the engine ignores it when blank_word is empty.

Example ID question:
{
  "id": "is-01",
  "sentence": "dīcō puerōs in forō esse.",
  "blank_word": "",
  "display": "dīcō puerōs in forō esse.",
  "translation": "I say that the boys are in the forum.",
  "options": ["indirect statement", "purpose clause", "indirect command", "ablative absolute"],
  "correct": "indirect statement",
  "explanation": "After a verb of speaking/thinking (dīcō), the accusative + infinitive construction signals indirect statement: puerōs (accusative subject) + esse (infinitive)."
}

Example Translation question:
{
  "id": "is-06",
  "sentence": "crēdō mīlitēs fortiter pugnāvisse.",
  "blank_word": "",
  "display": "crēdō mīlitēs fortiter pugnāvisse.",
  "translation": "",
  "options": [
    "I believe the soldiers fought bravely.",
    "I order the soldiers to fight bravely.",
    "The soldiers believe they fought bravely.",
    "I believe the soldiers are fighting bravely."
  ],
  "correct": "I believe the soldiers fought bravely.",
  "explanation": "pugnāvisse is the perfect infinitive (pugnāvisse = to have fought), placing the action before the main verb. mīlitēs is accusative as the subject of the indirect statement."
}

**gcse-purpose.json** (id prefix: pur-):
- Construction: ut + subjunctive (positive), nē + subjunctive (negative); also ad + gerund/gerundive for purpose
- Key signal: verb of motion/action + ut + subjunctive expressing aim
- ID options pool: "purpose clause", "result clause", "indirect command", "fearing clause"
- 5 ID sentences (mix positive/negative purpose) + 5 translation sentences
- Important explanatory contrast: purpose asks "why?"; result describes what happened

**gcse-result.json** (id prefix: res-):
- Construction: tam/ita/adeo/tantus + ut + subjunctive (note: same ut but result has an intensifying word)
- Key signal: tam, ita, adeō, tantus in the main clause + ut + subjunctive
- ID options pool: "result clause", "purpose clause", "temporal clause", "indirect statement"
- 5 ID sentences + 5 translation sentences

**gcse-fearing.json** (id prefix: fear-):
- Construction: verb of fearing (timeō, vereor, metuō) + nē (positive fear) or ut (negative fear)
- Key signal: nē after a fear verb = positive fear ("I fear that..."); ut after fear verb = negative fear ("I fear that...not")
- ID options pool: "fearing clause", "purpose clause", "result clause", "indirect command"
- 5 ID sentences + 5 translation sentences

**gcse-ablative-absolute.json** (id prefix: aa-):
- Construction: noun/pronoun + participle, both in ablative, grammatically independent of main clause
- Key signal: ablative noun + ablative participle phrase with no grammatical connection to main verb's subject
- ID options pool: "ablative absolute", "participial phrase", "temporal clause", "purpose clause"
- 5 ID sentences + 5 translation sentences
- Representative constructions: rege occiso, duce profecto, his rebus auditis, militibus discessis

**gcse-temporal-causal-concessive.json** (id prefix: tcc-):
- Construction: cum + indicative (temporal: when); cum + subjunctive (causal: since, or concessive: although); quamquam + indicative (concessive: although); dum + indicative (while)
- Key signal table (encode in explanations): cum + indicative = temporal; cum + subjunctive imperfect/pluperfect = causal or concessive (context determines); quamquam = concessive
- ID options pool: "temporal clause", "causal clause", "concessive clause", "purpose clause"
- 5 ID sentences (vary cum + indicative vs cum + subjunctive vs quamquam) + 5 translation sentences

**gcse-conditionals.json** (id prefix: cond-):
- Construction: si/nisi + indicative (open/simple); si + subjunctive (unfulfilled present = imperfect subjunctive; unfulfilled past = pluperfect subjunctive)
- Key signal: si/nisi; then tense of subjunctive signals open vs unfulfilled
- ID options pool: "open conditional", "unfulfilled conditional (present)", "unfulfilled conditional (past)", "temporal clause"
- 5 ID sentences + 5 translation sentences

Same quality bar as Task 1: every explanation names the signal, construction, and translation strategy.
